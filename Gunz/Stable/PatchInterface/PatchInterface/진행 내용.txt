- 기능 -
Keeper에 접속을 하여 Server의 시작, 정지를 제어함.(X)
Keeper에 접속을 하여 Keeper를 통해서 Server의 정보를 가져옴.(X)
Keeper에 접속을 하여 Server패치 작업을 요청을 함. (O)
Keeper에 접속을 하여 환경 설정을 함.(X)



- 완료된 작업 내용 -
Keeper에 접속을 하여 패치 작업명령을 전달 및 결과 내용을 받음.
Keeper에 접속을 하여 현제 Server의 현제 상태를 표시해 줌.
현제는 완료 버튼이 Keeper에 접속을 하여 패치 작업을 함. 시행하기전에 접속된 Keeper가 있으면 우선 연결을 종료함.
종료버튼은 연결 종료및 리스스를 릴리즈하고 프로그램을 종료시킴.



- 작업을 하기전에 반드시 봐야할 내용 - 
패치 작업을 단계적인 제어를 할수있게 인터페이스 수정이 필요함. 현제는 패치 작업을 수행하면 서버 종료부터 패치 서버시작까지 모두 한번에 해버림.
Keeper에 저장된 환경 설정 내용을 보여주거나 설정할수 있는 인터페이스가 필요함.



- Keeper Manager사용 방법 -
.\KeeperList.txt에 접속할 Keeper의 주소와 해당 주소를 Keeper Manager에 출력할 이름을 적어 놓으면,
	연결과 상태 출력에 사용함. 서버이름과 서버주소는 공백으로 구분을 하기에 구분할 경우에만 공백을 사용해아함.
	서버 리스트는 한줄에 하나의 서버내용만 입력함.
	- 내용 -
	[서버이름] [서버주소]
	ex. server1 192.168.0.31	( o )
		server 1 192.168.0.31	( x )
